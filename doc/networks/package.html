1 The networks Software

1.1 graph

The graph package actually contains two submodules: graph and search. The former contains utilities for manipulating a graph, while the latter contains utilities for searching a graph.

1.1.1 graph

 (require networks/graph/graph) package: networks

graph is a utility collection that handles the definition of basic networks. There are two components: graph creation and graph searching. Networks are represented as a list of nodes.

struct

(struct node (label edges))

  label : symbol?
  edges : (setof (all/c symbol?))
The node is the core structure used in this package. Every node has a label, which must be unique in whatever networks this node appears in. The node’s edges will be stored as a set internally. The symbols in the edges should refer to labels of nodes in the network. When creating a node, the edges may be passed in as a list, in which case the list will be converted to a set using list->set.

For example, here is a representation of the early 13-node ARPA network:

Examples:

> (define SRI  (node 'sri  '(utah stan ucla ucsb)))
> (define UCSB (node 'ucsb '(sri  ucla)))
> (define UCLA (node 'ucla '(ucsb stan  sri rand)))
> (define STAN (node 'stan '(sri  ucla)))
> (define UTAH (node 'utah '(sri  sdc  mit)))
> (define SDC  (node 'sdc  '(utah rand)))
> (define RAND (node 'rand '(ucla bbn sdc)))
> (define MIT  (node 'mit  '(utah bbn  linc)))
> (define BBN  (node 'bbn  '(rand mit  harv)))
> (define HARV (node 'harv '(bbn  carn)))
> (define CARN (node 'carn '(harv case)))
> (define CASE (node 'case '(carn linc)))
> (define LINC (node 'linc '(case mit)))
> (define ARPA-NET
    (list SRI
          UCSB
          UCLA
          STAN
          UTAH
          SDC
          RAND
          MIT
          BBN
          HARV
          CARN
          CASE
          LINC))

procedure

(node-label node) → (symbol?)

  node : node?
node-label returns the label for the node, which is a symbol. This identifier should be used in referring to the node in the edges of other nodes.

Example:

> (node-label MIT)

'mit

procedure

(node-edges node) → (setof (all/c symbol?))

  node : node?
node-edges returns a list of the node’s edges; these will all be symbols that should refer to nodes in the same network.

Example:

> (node-edges MIT)

(set 'utah 'bbn 'linc)

procedure

(node? id) → (boolean?)

  id : any/c
node? is a predicate returning true if the identifier names node.

Examples:

> (node? MIT)

#t

> (node? (node-label MIT))

#f

procedure

(neighbours? from to) → (boolean?)

  from : node?
  to : node?
neighbours? is a predicate that returns true if to’s label appears in from’s edges.

Examples:

> (define node-1 (node 'node-1 '(node-2 node-3)))
> (define node-2 (node 'node-2 '(node-1 node-3)))
> (define node-3 (node 'node-3 '(node-2)))
> (neighbours? node-1 node-2)

#t

> (neighbours? node-2 node-1)

#t

> (neighbours? node-1 node-3)

#t

> (neighbours? node-3 node-1)

#f

procedure

(path? path) → (boolean?)

  path : (listof (all/c node?))
path? returns true if an path exists between the nodes in the network. That is, each node is a neighbour of the next node in the list (excluding the last node, which ends the path.)

Examples:

> (path? (list node-1 node-2 node-3))

#t

> (path? (list node-1 node-3 node-2))

#t

> (path? (list node-3 node-1 node-2))

#f

procedure

(simple-path? path) → (boolean?)

  path : (listof (all/c node?))
A path is said to be a simple path when no node appears more than once in the path.

Examples:

> (path? (list node-1 node-2 node-1 node-3))

#t

> (simple-path? (list node-1 node-2 node-1 node-3))

#f

> (path? (list node-1 node-2 node-3))

#t

> (simple-path? (list node-1 node-2 node-3))

#t

procedure

(cycle? path) → (boolean?)

  path : (listof (all/c node?))
A cycle exists when a path starts and ends on the same node.

Examples:

> (cycle? (list node-3 node-2 node-1 node-3))

#t

> (cycle? (list node-1 node-2 node-3))

#f

procedure

(link/directed from to) → (node?)

  from : node?
  to : node?
link/directed creates an edge between from and to; as it is a directed link, it will only be created one-way. Linking is non-destructive, and returns a new node with to in its set of edges. If the two nodes were already linked, nothing happens.

Examples:

> (node-edges MIT)

(set 'utah 'bbn 'linc)

> (link/directed MIT UTAH)

(node 'mit (set 'utah 'bbn 'linc))

> (link/directed MIT HARV)

(node 'mit (set 'utah 'bbn 'linc 'harv))

> (neighbours? MIT HARV)

#f

> MIT

(node 'mit (set 'utah 'bbn 'linc))

procedure

(link/undirected from to) → 
node? node?
  from : node?
  to : node?
link/undirected creates an edge between from and to; as it is an undirected link, it will be created both ways. Accordingly, link/undirected returns a pair of new nodes. If the two nodes were already linked, nothing happens. Like link/directed, it is non-destructive.

Examples:

> (node-edges MIT)

(set 'utah 'bbn 'linc)

> (node-edges UTAH)

(set 'sri 'sdc 'mit)

> (node-edges HARV)

(set 'bbn 'carn)

> (link/undirected MIT UTAH)

(node 'mit (set 'utah 'bbn 'linc))

(node 'utah (set 'sri 'sdc 'mit))

> (link/undirected MIT HARV)

(node 'mit (set 'utah 'bbn 'linc 'harv))

(node 'harv (set 'mit 'bbn 'carn))

> (neighbours? MIT HARV)

#f

> MIT

(node 'mit (set 'utah 'bbn 'linc))

procedure

(get-node network label) → (or/c node? void?)

  network : (listof (all/c node?))
  label : symbol?
get-node finds the labeled node in the network. It returns void if no node with the label exists in the network.

Examples:

> (get-node ARPA-NET 'mit)

(node 'mit (set 'utah 'bbn 'linc))

> (get-node ARPA-NET 'rpi)

#<procedure:void>

procedure

(get-node-map network) → (lambda node?)

  network : (listof (all/c node?))
get-node-map returns a closure over get-node and the network. This is a convience for use in map calls.

Example:

> (set-map (node-edges MIT)
           (get-node-map ARPA-NET))

(list

 (node 'linc (set 'case 'mit))

 (node 'bbn (set 'rand 'mit 'harv))

 (node 'utah (set 'sri 'sdc 'mit)))

procedure

(followers network node)

 → (and/c integer? (or/c zero? positive?))
  network : (list-of (node?))
  node : node?
followers returns a count of edges pointing into the node. If the network is undirected, this count will be the same as the count returned from following.

Examples:

> (define simple-network (list node-1 node-2 node-3))
> (followers simple-network node-3)

2

> (followers simple-network node-1)

1

procedure

(following network node)

 → (and/c integer? (or/c zero? positive?))
  network : (list-of (node?))
  node : node?
following returns a count of edges leading out of the node. If the network is undirected, this count will be same as the count returned from followers.

Examples:

> (following simple-network node-3)

1

> (following simple-network node-1)

2

procedure

(sort-by-followers network)

 → 
(list-of
 (pair? (and/c integer? (or/c zero? positive?))
        node?))
  network : (list-of (node?))
sort-by-followers returns a list of nodes sorted in descending order by most followers. It is non-destructive, and list returned

Examples:

> (define test-network
    (list
     (node 'node-1 '())
     (node 'node-2 '(node-1 node-3))
     (node 'node-3 '(node-1 node-2))
     (node 'node-4 '(node-3))
     (node 'node-6 '())
     (node 'node-5 '(node-1))))
> (sort-by-followers test-network)

(list

 (cons 3 (node 'node-1 (set)))

 (cons 2 (node 'node-3 (set 'node-1 'node-2)))

 (cons 1 (node 'node-2 (set 'node-1 'node-3)))

 (cons 0 (node 'node-4 (set 'node-3)))

 (cons 0 (node 'node-6 (set)))

 (cons 0 (node 'node-5 (set 'node-1))))

This can be useful in identifying key communications hubs, for example:

Example:

> (sort-by-followers ARPA-NET)

(list

 (cons 4 (node 'sri (set 'utah 'stan 'ucla 'ucsb)))

 (cons 4 (node 'ucla (set 'rand 'stan 'ucsb 'sri)))

 (cons 3 (node 'utah (set 'sri 'sdc 'mit)))

 (cons 3 (node 'rand (set 'ucla 'sdc 'bbn)))

 (cons 3 (node 'mit (set 'utah 'bbn 'linc)))

 (cons 3 (node 'bbn (set 'rand 'mit 'harv)))

 (cons 2 (node 'ucsb (set 'ucla 'sri)))

 (cons 2 (node 'stan (set 'ucla 'sri)))

 (cons 2 (node 'sdc (set 'rand 'utah)))

 (cons 2 (node 'harv (set 'bbn 'carn)))

 (cons 2 (node 'carn (set 'case 'harv)))

 (cons 2 (node 'case (set 'linc 'carn)))

 (cons 2 (node 'linc (set 'case 'mit))))

One might surmise by looking at the above rankings that UCLA and SRI were core hubs in the ARPA network.

directed-network? returns true if the network is directed; that is, every edge coming into a node is matched by a corresponding edge outward.

Examples:

> (directed-network? ARPA-NET)

> (directed-network? test-network)

1.1.2 search

 (require networks/graph/search) package: networks

The search package contains functions for searching through a network.

procedure

(bfs network start end) → 
(or/c
 (and/c (list-of node?) path?)
 (empty?))
  network : (listof (node?))
  start : symbol?
  end : symbol?
bfs performs a breadth-first search for a path between the two nodes in the network. However, if no path can be found, an empty list will be returned. It will perform a basic best-effort attempt at building a simplified path from the returned path. This path minimisation can be useful; prior to implementing it, the bfs in the examples below included extraneous edges, such as going from 'sri to 'ucsb to 'ucla.

Examples:

> (define bfs-sri->harv (bfs ARPA-NET 'sri 'harv))
> bfs-sri->harv

(list

 (node 'sri (set 'utah 'stan 'ucla 'ucsb))

 (node 'ucla (set 'rand 'stan 'ucsb 'sri))

 (node 'rand (set 'ucla 'sdc 'bbn))

 (node 'bbn (set 'rand 'mit 'harv))

 (node 'harv (set 'bbn 'carn)))

> (path? bfs-sri->harv)

#t

> (simple-path? bfs-sri->harv)

#t

procedure

(dfs network start end) → 
(or/c
 (and/c (list-of node?) path?)
 (empty?))
  network : (listof (node?))
  start : symbol?
  end : symbol?
dfs performs a depth-first search for a path between the two nodes in the network. However, if no path can be found, an empty list will be returned. It will perform a basic best-effort attempt building a simplified path from the returned path, as with bfs.

Examples:

> (define dfs-sri->harv (dfs ARPA-NET 'sri 'harv))
> dfs-sri->harv

(list

 (node 'sri (set 'utah 'stan 'ucla 'ucsb))

 (node 'ucla (set 'rand 'stan 'ucsb 'sri))

 (node 'rand (set 'ucla 'sdc 'bbn))

 (node 'bbn (set 'rand 'mit 'harv))

 (node 'harv (set 'bbn 'carn)))

> (path? dfs-sri->harv)

#t

> (simple-path? dfs-sri->harv)

#t

procedure

(connected? network) → (boolean?)

  network : (list-of node?)
A network is said to be connected if for every pair of nodes in the network, there exists a path between them. An example is the ARPA network: every system can get a message to any other system in the network. This function is current implemented by running bfs on every pair of nodes in the network; it isn’t a very bright means to accomplish this, but it works. For larger networks, it will certainly be less performant.

Examples:

> (connected? ARPA-NET)

#t

> (define disconnected-network
    (list
     (node 'node-1 '(node-2 node-3))
     (node 'node-2 '(node-1 node-3))
     (node 'node-3 '(node-1))
     (node 'node-4 '(node-5))
     (node 'node-5 '(node-4))))
> (connected? disconnected-network)

#f

procedure

(clean-path path) → (list-of (node?))

  path : (list-of (node?))
clean-path will attempt to find the shortest path that goes from the first node in the path to the final node. It does so by recursing through the list, skipping to the farthest neighbour in the path it can find.

Example:

> (let ([path (list
               (node 'sri (set 'utah 'stan 'ucla 'ucsb))
               (node 'ucsb (set 'ucla 'sri))
               (node 'ucla (set 'rand 'stan 'ucsb 'sri))
               (node 'rand (set 'bbn 'ucla 'sdc))
               (node 'bbn (set 'rand 'mit 'harv))
               (node 'harv (set 'bbn 'carn)))])
    (clean-path path))

(list

 (node 'sri (set 'utah 'stan 'ucla 'ucsb))

 (node 'ucla (set 'rand 'stan 'ucsb 'sri))

 (node 'rand (set 'ucla 'sdc 'bbn))

 (node 'bbn (set 'rand 'mit 'harv))

 (node 'harv (set 'bbn 'carn)))
